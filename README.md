# 自动化截流 · MVP（抖音 + 小红书）

把「找链接 → 去重 → 打分排序 → 排队 → 人工辅助发布 → 记录复盘」跑成一个最小闭环。
纯 Python 标准库 + CSV，**无需安装任何依赖**，无后台。

```
关键词库 → 采集(MediaCrawler/手动) → 去重·打分·排序 → 待处理队列 → 人工辅助发布 → 历史·复盘
```

## 快速开始

```bash
./run.sh demo      # 用样例数据跑通打分，生成 data/queue.csv
./run.sh web       # 打开本地 Web 控制台（推荐）
# 或
./run.sh publish   # 命令行版人工辅助发布
./run.sh report    # 复盘统计
./run.sh test      # 自测（全绿即逻辑正常）
```

> 不用 `run.sh` 也行，直接 `python3 score.py` / `python3 serve.py` 等。

## 文件总览

| 文件 | 作用 |
|------|------|
| `config.json` | 关键词分级、排除词、打分权重、作者冷却天数（**最常调的就是这个**） |
| `keywords.txt` | 给采集端用的搜索词 |
| `comments.json` | 多版本评论模板（每条强制填针对该帖的 `{hook}`） |
| `score.py` | **核心**：去重 + 打分 + 排序 → `data/queue.csv` |
| `add_lead.py` | 手动录入线索（不接采集器也能当天起步） |
| `publish_assist.py` | 命令行：人工辅助发布（开链接 + 复制草稿 + 标状态） |
| `serve.py` | 本地 Web 控制台（同上，界面版，推荐） |
| `report.py` | 复盘统计（按状态/平台/优先级，发布率） |
| `selftest.py` | 自测：验证打分/去重/排除/冷却 |
| `store.py` | 共享数据层（字段/读写/状态流转） |
| `collectors/` | 采集层：MediaCrawler 适配器 + 说明 |
| `data/` | `leads.csv`(输入) / `queue.csv`(待办) / `history.csv`(记录) |

## 每天怎么跑

```bash
# 1) 拿线索：接 MediaCrawler（见 collectors/README.md）或 python3 add_lead.py 手动录
# 2) 去重 + 打分 + 排序
python3 score.py
# 3) 从高优先级往下，人工把关发布（二选一）
python3 serve.py            # Web 控制台
python3 publish_assist.py   # 命令行
# 4) 隔段时间复盘
python3 report.py
```

发布流程：工具帮你**打开链接 + 把评论草稿复制到剪贴板**，**你**粘贴、改成针对该帖的真实回复、自己点发送，然后标 `posted / skipped / failed`。

## 排序逻辑（最值钱的部分）

`分数 = 关键词分级基分 + 意图加分 + 时效加分 + 评论活跃加分`，命中排除词直接丢弃。

- **高**：搬家回国 / 家具海运 / 大件托运 / 清关 / 双清包税 / 钢琴托运 …
- **意图信号（最高价值，常在评论区）**：怎么寄 / 多少钱 / 清关麻烦吗 / 包税吗 / 求推荐 …
- **排除**：招聘 / 加盟 / 培训 / 骗局 / 求职 …
- **去重**：内容 ID + 链接去重；同一作者 14 天内不重复触达（`config.json` 可调）。

## 边界（刻意为之）

这套**只做**发现 / 去重 / 排序 / 人工辅助发布。
**不做**自动点发送、多账号群控、绕风控——那是负 ROI 的封号跑步机，且违反平台 ToS。
最后那一下由人来点：成本最低的“反风控”，转化也更高。

## 合规提示

- 采集只取**公开**数据、控制频率、遵守各平台 ToS；用于发现 + 人工跟进。
- 评论务必每条不同、针对该帖，别一字不差刷模板。
