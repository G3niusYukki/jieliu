# 自动化截流 · 找链接（抖音 + 小红书）

**一个脚本**：采集 → 打分筛选 → 输出「需人工手动截流的链接清单」，并可选用 AI 生成评论草稿。
只负责**找 + 写草稿**，**不自动发布**——审核和发布是人工的事。输出过的链接**永久排除**，每次跑只给你全新的。

业务逻辑纯 Python 标准库，无需 pip 装额外依赖（仅采集器 MediaCrawler 需一次性安装）。

## 用法（只有一个脚本 `jieliu.py`）

```bash
python3 jieliu.py setup                   # ① 一次性：装好采集器（clone + 依赖 + 补丁 + 自检）
python3 jieliu.py                          # ② 采集抖音+小红书 → 输出新链接到 data/截流链接.csv
python3 jieliu.py serve                    # ③（可选）开网页看板给运营用：看/筛/点链接·标记已截流·跑采集·导出
```

第 ①②步是命令行流程；想给运营一个网页用，跑第 ③步（见下「给运营用：网页看板」）。日常只需重复第 ②或③步。常用选项：

```bash
python3 jieliu.py --platform xhs --max 5   # 只采小红书，每平台最多 5 帖
python3 jieliu.py --get-comment false      # 不抓评论（更快更稳，但少了评论区买家）
python3 jieliu.py --max-comments 10        # 每帖最多抓 10 条评论（降请求量=降封控）
python3 jieliu.py --no-crawl               # 跳过采集，用上次抓到的数据重新筛
python3 jieliu.py --reset-seen             # 清空「已输出」记录（之后会重新给出旧链接）
python3 jieliu.py --selftest               # 自检（不联网、不动你的数据）
```

> 采集那一步会自动拉起浏览器让你**扫码登录**——请用**专门的小号**，别用你将来发评论触达客户的账号。
> 本脚本不碰账号密码、不自动登录、不发布。

## 输出长什么样

`data/截流链接.csv`（按优先级从高到低排好），每行 = 一条待你手动去截流的链接 + 一行上下文：

| 列 | 含义 |
|----|------|
| `score` / `priority` | 分数 / 高中低优先级（从上往下做）|
| `platform` | 抖音 / 小红书 |
| `type` | `帖子`（去帖子下评论）或 `评论者`（直接回复评论区那个问价的人，转化更高）|
| `intent` | 命中的买家意图（多少钱 / 怎么寄 / 求推荐 …）|
| `who` | 帖主昵称 / 要回复的评论者昵称 |
| `link` | **要打开去截流的链接** |
| `what` | 帖子标题 / 该评论者问了啥 |

控制台也会打印前 20 条，方便你直接点。

## 给运营用：网页看板（`serve`）

不想让运营碰命令行，就开个本地网页看板：

```bash
python3 jieliu.py serve                 # 默认 http://127.0.0.1:8787（启动后自动打开浏览器）
python3 jieliu.py serve --host 0.0.0.0 --port 8787   # 让同一局域网的运营也能访问
python3 jieliu.py serve --no-open       # 不自动打开浏览器（部署在服务器/远程时）
```

**更省事 · 一键启动**（自动起服务 + 开浏览器）：
- **Windows**：在项目文件夹里输入 `kanban`（或双击 `kanban.bat`）。
- **macOS / Linux**：在项目文件夹里运行 `./kanban`。

> 首次启动会**自动安装采集器**（clone + 依赖 + 补丁，几分钟，仅一次），装好后才打开看板。安装是**纯 Python、跨平台**，只需本机有 **git + Python**（Windows 无需 bash）；装不动也不影响看板的看/筛/标记/导出/AI，只是「跑采集」要等装好。`--no-setup` 可关掉自动安装。

浏览器打开后，运营在网页上即可：

- **填业务 + 历史评论模板**（页面顶部两个输入框）：业务用来生成搜索关键词，模板用来生成风格一致的评论草稿；保存后长期留着；
- **用业务生成关键词**：点「✨ 用业务生成关键词」，AI 据业务产出消费者大白话搜索词，自动填进关键词框，检查后保存即可采集；
- **看 / 筛 / 点链接**：按平台、类型（帖子/评论者）、优先级筛，搜「找谁/意图/内容」，点「打开」直达原帖；
- **AI 写评论草稿**：点某条的「写评论」，结合 业务 + 模板 + 该帖内容 生成一条评论草稿（可编辑、可复制）；**只是草稿，请人工审核后手动去原帖发布**；
- **标记『已截流』**：处理完打个勾（状态存 `data/worklist.json`），默认隐藏已截流，避免多人重复做同一条；
- **跑一次采集 / 停止 / 导出当前列表**：网页上点按钮就能触发采集（可设平台/帖数/评论数/抓不抓评论）；**首次会在这台机器弹出浏览器扫码，请用专门小号**。觉得跑够了点「■ 停止并出本次链接」，它会用**已采集到的数据**直接出链接（等于程序化 Ctrl+C，不浪费），完成后自动刷新看板。

看板是**累积工作清单**：每跑一次采集，新链接并进清单（已截流状态与评论草稿都保留），运营可以一直往下清。

### 开启 AI（生成关键词 / 评论草稿）— 火山方舟 ARK · BYOK

AI 走**火山方舟 ARK**（大陆端点，OpenAI 兼容，纯标准库 `urllib`，不加任何依赖）。
端点：`https://ark.cn-beijing.volces.com/api/v3/chat/completions`。**自带 Key（BYOK）**，两种方式任选：

**A. 环境变量（推荐，更安全，Key 不落盘）**
```bash
export ARK_API_KEY=你的Key
export ARK_MODEL=doubao-seed-1-6-250615   # 可选；填模型名或你的接入点 ep-xxxx
# export ARK_BASE_URL=...                  # 可选；自建/代理网关时才改
python3 jieliu.py serve
```

**B. 在看板里填**：打开看板 →「① 业务/…」面板底部「AI 设置 · 火山方舟 ARK」→ 粘贴你的 Key 和模型 → 保存（存在本地 `data/aiconfig.json`，不入库）。

> Key 哪来：火山引擎控制台 → 方舟 Ark →「API Key 管理」创建；模型在方舟「开通管理」里开通后，用模型名或创建的接入点 `ep-xxxx` 都行。
> 没配 Key 也能正常用看板其余功能，只是 AI 两个动作会提示去配 Key。生成的评论是**草稿**，发布与否、怎么发都由人工决定。
> 安全：看板 `GET /api/config` **不会回传完整 Key**（只给后四位提示）；用 `--host 0.0.0.0` 暴露给局域网时，优先用环境变量方式。

> ⚠️ `--host 0.0.0.0` 会让同局域网都能访问，且**没有登录验证**——只在可信内网这么开，别暴露到公网。采集仍只在本机进行，运营那头碰不到账号。

## 它怎么挑（最值钱的部分，在 `config.json` 调）

`分数 = 关键词分级 + 意图加分(强>弱，评论区×倍率) + 时效 + 评论活跃 + 海外属地`；命中下列直接丢弃：

- **排除**：招聘 / 加盟 / 培训 / 骗局 …
- **噪声丢弃**：货代知识/科普/FBA/B2B/国内大件，以及**作者本身是货代/集运卖家**
- **评论者线索**：评论区「问价/问怎么寄」的人会被单独抽出来（`type=评论者`），可直接回复 TA
- **海外属地**：发帖/评论者在 美/澳/加/英… → 加分（「海运回国真买家」最廉价的判别特征）
- **永久去重**：任一链接被输出过，以后不再出现（记录在 `data/seen.json`）

想把「对口买家」比例拉高，主要调 `keywords.txt`（搜索词，建议用消费者大白话：海运回国 / 留学回国行李 / 美国寄回国 …）和 `config.json`（分级与权重）。

## 文件总览

| 文件 | 作用 |
|------|------|
| `jieliu.py` | **唯一要运行的脚本**：采集 + 打分筛选 + 输出链接 + 永久去重 + 自检 |
| `config.json` | 关键词分级、意图词、排除/卖家/科普词、海外属地、打分权重（**最常调**）|
| `keywords.txt` | 采集用的搜索词 |
| `collectors/setup_mediacrawler.sh` | 一键装好可跑的采集器（被 `jieliu.py setup` 调用）|
| `collectors/patches/` | 让 MediaCrawler 真正跑通的补丁（CDP 自启 / xhshow 签名兼容）|
| `data/截流链接.csv` | **产物**：待人工截流的链接清单（不入库）|
| `data/seen.json` | 已输出过的链接记录，用于永久去重（不入库）|
| `data/worklist.json` | 看板的累积工作清单 + 『已截流』标记（不入库）|

## 频率与账号风控（务必看）

- 当前频率温和：MediaCrawler 串行（并发 1）、每个请求间隔 sleep 2 秒；默认每平台 5 帖、每帖 20 条评论，一次约几十个请求、两三分钟。**频率本身不是主要风险。**
- **最大的封号风险是「用错账号」**：平台能识别某账号正被自动化浏览器驱动，直接**账号级封禁**（手机端 + 电脑端同时失效）。所以**采集必须用专门的采集小号**（消耗品，被封就换），**绝不能用你真人/将来发评论触达客户的账号**去扫码。
- 进一步降风险：`--max` 调小（如 3）、别短时间反复跑、`--get-comment false` 更轻；抖音比小红书风控更狠。

## 国内网络环境（部署常见卡点）

**运行时不受影响**：采集打的是抖音/小红书、AI 打的是火山方舟 `ark.cn-beijing.volces.com`，都是境内。
**卡点全在「首次安装」的三个境外下载**，用国内镜像模式一条命令解决：

```bash
python jieliu.py setup --cn        # pip 走清华源；CDP 用系统 Chrome/Edge → 跳过 playwright chromium 下载
```

`--cn` 具体做了：
- **pip 依赖** → 清华源 `pypi.tuna.tsinghua.edu.cn`（也可用 `JIELIU_PIP_INDEX=<源>` 换别的）；
- **playwright chromium（~150MB，常被墙）** → **直接跳过**。CDP 模式用系统 **Chrome/Edge**（Windows 自带 Edge 即可，无需额外装浏览器）；
- **GitHub 克隆**（唯一仍依赖 GitHub 的一步）：拉不动就二选一——
  - `JIELIU_MC_REPO=<镜像/加速地址> python jieliu.py setup --cn`（指向 GitHub 镜像或加速）；
  - 或手动下载 MediaCrawler 解压到 `vendor/MediaCrawler`（含 `main.py`），再跑 setup（会自动跳过克隆）。

> 想让 `kanban` / `serve` 的**首次自动安装**也走镜像：先设环境变量 `JIELIU_CN=1`（Windows: `set JIELIU_CN=1`）再启动；或先手动 `python jieliu.py setup --cn` 装一次，之后 `kanban` 直接用。

## 在 Windows 上运行

- **跑看板/采集/停止**：均已适配 Windows（venv 路径、停止采集用 `CTRL_BREAK` 触发抢救出链接）。一键启动用 `kanban`（或双击 `kanban.bat`）。
- **首次装采集器**：`kanban` / `serve` 首次启动会自动安装，安装是**纯 Python、跨平台**，只要本机有 **git 和 Python**（Windows **无需 bash**）。手动安装：`python jieliu.py setup`。装完后日常用 `kanban` 即可。
  > `collectors/setup_mediacrawler.sh` 是等价的 bash 版（Unix 备选）；Windows/默认都走 `python jieliu.py setup`。
- 采集要本机装有 Chrome；首次扫码登录请用**专门小号**。

## 边界与合规

- 只做**发现**：找链接、打分、去重。**不做**回复、自动发送、多账号群控、绕风控（AI 只生成草稿，发布由人工）。
- 只采**公开**数据、控制频率、用专门小号；评论原文落盘前已**脱敏**手机号/微信/QQ/邮箱。
- 不存他人手机号/微信号等敏感联系方式；数据仅本地自用、不转卖。回复务必每条不同、针对该帖/该人。
